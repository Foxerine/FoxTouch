# FoxTouch

[English](README.md) | [简体中文](README_zh-CN.md) | [繁體中文](README_zh-TW.md) | **日本語** | [한국어](README_ko.md) | [Español](README_es.md) | [Bahasa Melayu](README_ms.md)

FoxTouch は [Claude Code](https://docs.anthropic.com/en/docs/claude-code) にインスパイアされた、AI 駆動の Android スマートフォンエージェントです。アクセシビリティサービスを通じて画面を観察し、UI 要素を理解し、自然言語の指示に基づいてユーザーの代わりに操作を実行します — AI コーディングアシスタントのエージェント体験をスマートフォンの画面に。

> **ステータス：概念実証 / 開発中**
>
> このプロジェクトは初期実験段階にあります。機能は不完全、不安定、または大幅に変更される可能性があります。

### ⚠️ 重要な注意事項

**AI 生成コード**：このプロジェクトのコードの大部分は AI（Claude）によって生成され、人間が方向性の指示とレビューを行っています。アプリケーションアーキテクチャ、UI 実装、ツールシステム設計、アクセシビリティ統合、システムプロンプトを含みます。

### 仕組み

1. ユーザーが自然言語でタスクを説明
2. FoxTouch が Android アクセシビリティ API で画面を読み取り
3. LLM（設定可能）が UI を分析し、アクションを決定
4. FoxTouch がアクセシビリティサービスで操作を実行（タップ、入力、スクロール、スワイプなど）
5. 「観察→思考→実行」のサイクルをタスク完了まで繰り返し

### ユースケース

**日常の自動化**

> **あなた：** 「出前を注文して、3000円くらいで、昨日伝えた味の好みに合わせて、一番お得なクーポンを使って」
>
> FoxTouch がメモリから味の好みを思い出す → 出前アプリを開く → レストランを閲覧 → 予算と好みに合った注文を計画 → 最適なクーポンを適用 → 支払い前に確認を求める

> **あなた：** 「お母さんに LINE で帰りが遅くなると伝えて、明朝7時のアラームも設定して」
>
> FoxTouch が LINE を開く → お母さんを見つける → メッセージを入力・送信 → 時計を開く → アラームを作成 → 両方のタスク完了を確認

**情報検索・整理**

> **あなた：** 「Amazon と楽天で AirPods Pro の価格を比較して、どっちが安いか教えて」
>
> FoxTouch が Amazon を開く → 検索して価格を記録 → 楽天を開く → 検索して価格を記録 → 比較結果を報告

> **あなた：** 「この画面の内容を英語に翻訳して」
>
> FoxTouch が現在の画面内容を読み取る → チャットで完全な翻訳を提供

**SNS 操作**

> **あなた：** 「最新の写真を X にポストして、夕焼けについてのキャプションをつけて」
>
> FoxTouch が X を開く → 新規投稿 → 最新の写真を選択 → キャプションを作成 → 投稿前に確認を求める

**デイリーチェックイン & 通知**

> **あなた：** 「原神のデイリーチェックインをやって」
>
> FoxTouch がゲームを開く → チェックインページに移動 → デイリー報酬を受け取る → 次回のために再利用可能なスキルとして保存

> **あなた：** 「未読通知を全部確認して、重要なものがあるか教えて」
>
> FoxTouch が通知シェードを開く → 各通知を読む → 重要なものをフィルタリングして要約 → 残りをクリア

### 主な機能

- **複数 LLM 対応** — OpenAI、Anthropic Claude、Google Gemini、OpenRouter、および OpenAI 互換 API
- **画面理解** — UI 要素ツリー解析 + 座標グリッドと要素ラベル付きのスクリーンショット注釈
- **フルデバイス操作** — タップ、入力、スクロール、スワイプ、長押し、ピンチ、戻る/ホーム、アプリ起動
- **プランモード** — 複雑なタスクを事前に計画し、ユーザー承認後に実行
- **スキルシステム** — アクションプランを保存し再利用
- **タスク追跡** — マルチステップの進捗をリアルタイム表示
- **フローティングオーバーレイ** — どのアプリからでもエージェントを制御
- **内蔵 IME** — 全アプリで確実なテキスト入力とクリップボードアクセスを実現する不可視入力方式
- **拡張スクリーンショット** — アクセシビリティスクリーンショットをブロックするアプリ向け MediaProjection フォールバック
- **コンテキスト圧縮** — トークン制限内に収めるための会話自動要約
- **音声入力 & 読み上げ** — 音声対話 + 応答の読み上げ
- **永続メモリ** — セッションをまたいで指示とコンテキストを記憶
- **安全機能** — ツールごとの承認、リスクレベル権限、YOLO モード切替
- **多言語 UI** — English、简体中文、繁體中文、日本語、한국어、Español、Bahasa Melayu

### 設計の詳細

<details>
<summary><b>コンテキスト自動圧縮</b> — トークン上限に近づくと会話を自動要約</summary>

長時間タスクが LLM のコンテキストウィンドウを使い切ると、FoxTouch はクラッシュやコンテキスト喪失の代わりに会話を自動圧縮します。

**段階的閾値** — 小さいコンテキストウィンドウほど早く圧縮（≤128K で 70%）、大きいウィンドウはより多く使用可能（>500K で 85%）。100 以上のモデルの内蔵データベースとランタイム API 発見による上書きをサポート。

**スマート要約** — 圧縮プロンプトは実用的な詳細の保持を要求：ユーザーメッセージの逐語記録、要素 ID、座標、パッケージ名、エラーパス、具体的な「次のステップ」。

**シームレスな継続** — 要約後、履歴をクリアし、現在のデバイスコンテキストを含む新しいシステムプロンプトを再注入。「中断がなかったかのように続行」という指示と共に要約をユーザーメッセージとして追加。

**別の圧縮モデル** — 要約に安価/高速なモデルを使用可能（例：Haiku で要約、Opus で実行）。

</details>

<details>
<summary><b>スクリーンショット注釈システム</b> — LLM の空間推論を支援する 4 層の視覚レイヤー</summary>

エージェントが画面を読み取る際、スクリーンショットに最大 4 つの注釈レイヤーをリクエスト可能：

1. **座標グリッド** — 200px ごとの参照線
2. **要素境界** — 色分け矩形：緑=クリック可能、青=スクロール可能、橙=編集可能、灰=その他。各 `[ID]` 付き
3. **テキスト/クラスラベル** — 要素のクラス名とテキスト内容
4. **クリックマーカー** — アクション後の検証、タップ位置を正確に表示

すべての座標は元の画面空間を使用 — 変換は内部で処理。

</details>

<details>
<summary><b>マルチプロバイダー LLM 抽象化</b> — 1 つのインターフェース、3 つの異なる API</summary>

すべてのプロバイダーが統一された `Flow<LLMEvent>` を発行しますが、内部では大きな API の違いを処理：

- **Claude** — Anthropic ネイティブ SSE ストリーミング。メッセージロールの正規化と連続同一ロールメッセージのマージ。拡張思考サポート。

- **Gemini** — ツール使用時は非ストリーミングを強制（`thoughtSignature` がストリーミングで不安定なため）。署名は後続メッセージで返送必須。

- **OpenAI** — オプションプロパティを nullable 型に変換し全プロパティを必須化。モデルが未使用パラメータに `null` を明示的に送信するよう強制。

</details>

<details>
<summary><b>3 段階テキスト入力</b> — カスケードフォールバックによるユニバーサル互換性</summary>

1. **内蔵 IME**（FoxTouchIME）— 不可視の InputMethodService。WebView、Flutter、カスタムコントロールに最も信頼性が高い。**Android 10+ のクリップボード読み取り制限を免除**。

2. **ACTION_SET_TEXT** — フォーカスノードへの標準アクセシビリティアクション。

3. **クリップボード貼り付け** — 最後の手段：クリップボードに書き込み → `ACTION_PASTE` で貼り付け。

IME の自動切替：起動 → テキストコミット → 前の入力方式に復帰、約 1.5 秒。

</details>

<details>
<summary><b>安全と権限システム</b> — 段階的リスクレベルとランタイムモード切替</summary>

各ツールがリスクレベル（低/中/高）を宣言。ユーザーは承認リクエストに対して：許可、常に許可、全て許可（現在のターンのみ YOLO モード）、拒否で応答可能。`confirm_completion` はタスク完了前に必ず確認。

</details>

<details>
<summary><b>プランモード</b> — 実行前の制限付き観察フェーズ</summary>

複雑なタスクでは、エージェントは読み取り専用ツールとプランニングツールのみ使用可能なプランモードに入ります。構造化されたプランを作成してユーザーレビューに提出、承認後にインタラクションツールをアンロック。プランは再利用可能なスキルとして保存可能。

</details>

<details>
<summary><b>オーバーレイアーキテクチャ</b> — システムオーバーレイサービスで Jetpack Compose をレンダリング</summary>

フローティングパネルは `ComposeView` + `TYPE_APPLICATION_OVERLAY`、カスタム `LifecycleOwner` でレンダリング。エージェント状態に応じてフォーカスモードを動的切替。スクリーンショット前にオーバーレイを非表示にし、50ms 待機後にキャプチャ。

</details>

<details>
<summary><b>クリップボードアクセス</b> — Android のクリップボード制限を 3 段階で回避</summary>

Android 10+ のクリップボード読み取り制限に対して：IME パス（制限免除）→ アクセシビリティサービスコンテキスト → 透明 Activity（一時的にウィンドウフォーカスを取得、2 秒以内に完了）。

</details>

# FoxTouch

[English](README.md) | [简体中文](README_zh-CN.md) | **繁體中文** | [日本語](README_ja.md) | [한국어](README_ko.md) | [Español](README_es.md) | [Bahasa Melayu](README_ms.md)

FoxTouch 是一個 AI 驅動的 Android 手機智慧體，靈感來自 [Claude Code](https://docs.anthropic.com/en/docs/claude-code)。它透過無障礙服務觀察螢幕、理解 UI 元素，並根據自然語言指令代替使用者執行操作 — 將 AI 程式助手的智慧體體驗帶到手機螢幕上。

> **狀態：概念驗證 / 開發中**
>
> 本專案處於早期實驗階段。功能可能不完整、不穩定，或發生重大變更。

### ⚠️ 重要聲明

**AI 生成程式碼**：本專案絕大部分程式碼由 AI（Claude）生成，由人類進行方向指導和審查。包括應用架構、UI 實作、工具系統設計、無障礙整合和系統提示詞。

**使用風險自負**：本軟體按「原樣」提供，不作任何保證。AI 智慧體可以在你的手機上執行點擊、輸入、滑動、啟動應用程式等操作。**你需要對使用本軟體產生的任何後果承擔全部責任。** 作者不對因使用本軟體可能導致的任何損害、資料遺失、意外購買或其他問題承擔責任。

### 運作原理

1. 使用者用自然語言描述任務
2. FoxTouch 透過 Android 無障礙 API 讀取螢幕
3. LLM（可設定）分析 UI 並決定操作
4. FoxTouch 透過無障礙服務執行操作（點擊、輸入、捲動、滑動等）
5. 「觀察-思考-執行」循環重複，直到任務完成

### 核心功能

- **多模型支援** — OpenAI、Anthropic Claude、Google Gemini、OpenRouter 及任何 OpenAI 相容 API
- **螢幕理解** — UI 元素樹解析 + 帶座標網格和元素標註的截圖
- **全面的裝置互動** — 點擊、輸入、捲動、滑動、長按、捏合、返回/主畫面導航、啟動應用程式
- **計畫模式** — 複雜任務先規劃再執行，使用者審核後才運行
- **技能系統** — 儲存並複用操作方案
- **任務追蹤** — 即時展示多步驟執行進度
- **懸浮視窗** — 透過底部懸浮列在任意應用程式中控制智慧體
- **內建輸入法** — 隱形輸入法，實現跨應用程式的可靠文字輸入和剪貼簿存取
- **增強截圖** — 對阻止無障礙截圖的應用程式（如微信）使用 MediaProjection 備選方案
- **上下文壓縮** — 自動摘要對話以保持在 token 限制內
- **語音輸入與朗讀** — 語音對話 + 朗讀回覆
- **持久記憶** — 智慧體跨工作階段記住指令和上下文
- **安全防護** — 逐工具審批、風險等級權限、YOLO 模式切換
- **多語言介面** — English、简体中文、繁體中文、日本語、한국어、Español、Bahasa Melayu

### 設計詳解

<details>
<summary><b>上下文自動壓縮</b> — 接近 token 上限時無縫摘要對話</summary>

當長任務填滿 LLM 上下文視窗時，FoxTouch 自動壓縮對話而非崩潰或遺失上下文。

**分層閾值** — 較小的上下文視窗更早觸發壓縮（≤128K 時在 70%），留出更多空間；較大視窗可填充更多（>500K 時 85%）。內建 100+ 模型上下文大小資料庫，支援執行時 API 發現覆寫。

**智慧摘要** — 壓縮提示詞要求保留可操作細節：逐字使用者訊息、元素 ID、座標、套件名稱、錯誤路徑和具體的「下一步」。工具參數截斷至 200 字元，結果截斷至 300 字元以過濾雜訊。

**無縫續接** — 摘要後清除歷史記錄，重新注入帶當前裝置上下文的系統提示詞，將摘要作為使用者訊息添加並指示「像中斷從未發生一樣繼續」。迴圈在當前輪次內恢復，使用者無感知。

**獨立壓縮模型** — 可使用更便宜/更快的模型進行摘要（如用 Haiku 摘要、Opus 執行），降低長任務成本。

</details>

<details>
<summary><b>截圖標註系統</b> — 四層獨立視覺標註輔助 LLM 空間推理</summary>

智慧體讀取螢幕時，可在截圖上請求最多四層標註：

1. **座標網格** — 每 200px 繪製參考線，提供絕對空間參考點
2. **元素邊界** — 彩色矩形框：綠色=可點擊、藍色=可捲動、橙色=可編輯、灰色=其他。每個標註 `[ID]`
3. **文字/類別標籤** — 元素類別名稱和文字內容顯示在邊界框下方
4. **點擊標記** — 操作後驗證，精確顯示點擊位置

所有座標使用原始螢幕空間（非縮放點陣圖空間）——轉換在內部完成。LLM 在絕對座標下推理，截圖則被縮放以節省 token。

</details>

<details>
<summary><b>多提供商 LLM 抽象</b> — 一個介面，三種截然不同的 API</summary>

所有提供商發出統一的 `Flow<LLMEvent>`，但各自在內部處理顯著的 API 差異：

- **Claude** — 使用 Anthropic 原生 SSE 串流。規範化訊息角色（工具結果變為使用者訊息），合併連續相同角色訊息以滿足 Claude 嚴格的交替要求。支援可設定預算的延伸思考。

- **Gemini** — 存在工具時強制非串流，因為 Gemini 的 `thoughtSignature`（工具呼叫必需）在串流模式下不可靠。簽章必須在後續訊息中回傳。

- **OpenAI** — 通過將可選屬性轉換為可空類型並使所有屬性成為必需，解決 OpenAI 填充所有 schema 屬性的問題。強制模型對未使用參數顯式發送 `null`。

</details>

<details>
<summary><b>三級文字輸入</b> — 級聯回退實現通用文字輸入相容</summary>

Android 上的文字輸入出人意料地不可靠。FoxTouch 實現了三級回退：

1. **內建輸入法**（FoxTouchIME）— 不可見的 InputMethodService，使用 `InputConnection.commitText()`。對 WebView、Flutter 應用和自訂控制項最可靠。同時**豁免 Android 10+ 剪貼簿讀取限制**。

2. **ACTION_SET_TEXT** — 對焦點節點的標準無障礙操作。適用於大多數原生 Android 文字框。

3. **剪貼簿貼上** — 最後手段：將文字寫入剪貼簿，然後透過無障礙分發 `ACTION_PASTE`。

輸入法自動切換：激活 → 提交文字 → 恢復使用者鍵盤，全程約 1.5 秒。

</details>

<details>
<summary><b>安全與權限系統</b> — 分級風險等級與執行時模式切換</summary>

每個工具宣告風險等級（低/中/高），使用者可對審批請求回應：允許、始終允許、全部允許（僅當前輪次 YOLO 模式）或拒絕。`confirm_completion` 確保智慧體在認為任務完成前總是詢問。

</details>

<details>
<summary><b>計畫模式</b> — 執行前的受限觀察階段</summary>

智慧體進入計畫模式後只能使用唯讀工具和計畫工具，不能執行任何狀態改變操作。將結構化計畫寫為 markdown 檔案供使用者審查，批准後才解鎖互動工具。計畫可儲存為可複用技能。

</details>

<details>
<summary><b>懸浮視窗架構</b> — 系統疊加層服務中渲染 Jetpack Compose</summary>

懸浮面板是 `ComposeView` + `TYPE_APPLICATION_OVERLAY`，由自訂 `LifecycleOwner` 渲染。根據智慧體狀態動態切換焦點模式。每次截圖前隱藏所有疊加層視圖，等待 50ms 後截圖，防止智慧體看到自己的 UI。

</details>

<details>
<summary><b>剪貼簿存取</b> — 三階段繞過 Android 剪貼簿限制</summary>

Android 10+ 限制剪貼簿讀取。FoxTouch 使用三種回退：IME 路徑（豁免限制）→ 無障礙服務上下文 → 透明 Activity（短暫獲取視窗焦點讀取剪貼簿，2 秒內完成）。

</details>

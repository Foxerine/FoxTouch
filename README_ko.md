# FoxTouch

[English](README.md) | [简体中文](README_zh-CN.md) | [繁體中文](README_zh-TW.md) | [日本語](README_ja.md) | **한국어** | [Español](README_es.md) | [Bahasa Melayu](README_ms.md)

FoxTouch는 [Claude Code](https://docs.anthropic.com/en/docs/claude-code)에서 영감을 받은 AI 기반 Android 스마트폰 에이전트입니다. 접근성 서비스를 통해 화면을 관찰하고, UI 요소를 이해하며, 자연어 지시에 따라 사용자를 대신하여 작업을 수행합니다 — AI 코딩 어시스턴트의 에이전트 경험을 스마트폰 화면으로.

> **상태: 개념 증명 / 개발 중**
>
> 이 프로젝트는 초기 실험 단계에 있습니다. 기능이 불완전하거나, 불안정하거나, 크게 변경될 수 있습니다.

### ⚠️ 중요 고지사항

**AI 생성 코드**: 이 프로젝트의 코드 대부분은 AI(Claude)에 의해 생성되었으며, 인간이 방향 지시와 검토를 수행했습니다. 애플리케이션 아키텍처, UI 구현, 도구 시스템 설계, 접근성 통합 및 시스템 프롬프트를 포함합니다.

**사용에 따른 위험은 본인 부담**: 이 소프트웨어는 어떠한 보증도 없이 "있는 그대로" 제공됩니다. AI 에이전트는 탭, 입력, 스와이프, 앱 실행 등 휴대폰에서 작업을 수행할 수 있습니다. **이 소프트웨어 사용으로 인한 모든 결과에 대해 사용자가 전적으로 책임집니다.** 저작자는 사용으로 인해 발생할 수 있는 손해, 데이터 손실, 의도하지 않은 구매 또는 기타 문제에 대해 어떠한 책임도 지지 않습니다.

### 작동 방식

1. 사용자가 자연어로 작업을 설명
2. FoxTouch가 Android 접근성 API로 화면을 읽음
3. LLM(설정 가능)이 UI를 분석하고 작업을 결정
4. FoxTouch가 접근성 서비스로 작업을 수행 (탭, 입력, 스크롤, 스와이프 등)
5. 작업이 완료될 때까지 "관찰-생각-실행" 사이클 반복

### 사용 사례

**일상 자동화**

> **당신:** "배달 음식 주문해줘, 3만원 정도로, 어제 알려준 내 취향대로 골라서, 가장 저렴한 쿠폰 써줘"
>
> FoxTouch가 메모리에서 취향을 불러옴 → 배달 앱 열기 → 음식점 탐색 → 예산과 취향에 맞는 주문 계획 → 최적 쿠폰 적용 → 결제 전 확인 요청

> **당신:** "엄마한테 카카오톡으로 늦게 간다고 보내고, 내일 아침 7시 알람 설정해줘"
>
> FoxTouch가 카카오톡 열기 → 엄마 찾기 → 메시지 입력 후 전송 → 시계 앱 열기 → 알람 생성 → 두 작업 완료 확인

**정보 검색 및 정리**

> **당신:** "쿠팡이랑 네이버에서 AirPods Pro 가격 비교해서 어디가 더 싼지 알려줘"
>
> FoxTouch가 쿠팡 열기 → 검색 후 가격 기록 → 네이버 열기 → 검색 후 가격 기록 → 비교 결과 보고

> **당신:** "이 화면 내용을 영어로 번역해줘"
>
> FoxTouch가 현재 화면 내용 읽기 → 채팅에서 전체 번역 제공

**소셜 미디어**

> **당신:** "최신 사진으로 인스타 스토리 올려줘, 노을 관련 캡션 달아서"
>
> FoxTouch가 Instagram 열기 → 스토리 생성 → 최신 사진 선택 → 캡션 작성 → 게시 전 확인 요청

**데일리 체크인 & 알림**

> **당신:** "원신 출석 체크 해줘"
>
> FoxTouch가 게임 열기 → 출석 페이지 이동 → 일일 보상 수령 → 다음에 재사용할 수 있는 스킬로 저장

> **당신:** "읽지 않은 알림 전부 확인해서 중요한 거 있으면 알려줘"
>
> FoxTouch가 알림 패널 열기 → 각 알림 읽기 → 중요한 항목 필터링 및 요약 → 나머지 정리

### 주요 기능

- **다중 LLM 지원** — OpenAI, Anthropic Claude, Google Gemini, OpenRouter 및 모든 OpenAI 호환 API
- **화면 이해** — UI 요소 트리 파싱 + 좌표 그리드와 요소 레이블이 포함된 스크린샷 주석
- **전체 기기 조작** — 탭, 입력, 스크롤, 스와이프, 길게 누르기, 핀치, 뒤로/홈, 앱 실행
- **계획 모드** — 복잡한 작업을 사전 계획 후 사용자 승인 후 실행
- **스킬 시스템** — 액션 플랜 저장 및 재사용
- **작업 추적** — 다단계 진행 상황 실시간 표시
- **플로팅 오버레이** — 모든 앱에서 에이전트 제어
- **내장 입력기** — 모든 앱에서 안정적인 텍스트 입력과 클립보드 접근을 위한 보이지 않는 입력 방식
- **향상된 스크린샷** — 접근성 스크린샷을 차단하는 앱을 위한 MediaProjection 폴백
- **컨텍스트 압축** — 토큰 한도 내 유지를 위한 대화 자동 요약
- **음성 입력 및 TTS** — 음성 대화 + 응답 읽어주기
- **영구 메모리** — 세션 간 지시사항과 컨텍스트 기억
- **안전 장치** — 도구별 승인, 위험 수준 권한, YOLO 모드 토글
- **다국어 UI** — English, 简体中文, 繁體中文, 日本語, 한국어, Español, Bahasa Melayu

### 설계 상세

<details>
<summary><b>컨텍스트 자동 압축</b> — 토큰 한도에 근접 시 대화를 자동 요약</summary>

장시간 작업이 LLM 컨텍스트 윈도우를 채우면, FoxTouch는 크래시나 컨텍스트 손실 대신 대화를 자동 압축합니다.

**계층적 임계값** — 작은 컨텍스트 윈도우는 더 일찍 압축 (≤128K에서 70%), 큰 윈도우는 더 많이 채울 수 있음 (>500K에서 85%). 100+ 모델의 내장 데이터베이스와 런타임 API 발견 오버라이드 지원.

**스마트 요약** — 압축 프롬프트가 실행 가능한 세부 사항 보존을 요구: 사용자 메시지 원문, 요소 ID, 좌표, 패키지 이름, 오류 경로, 구체적인 "다음 단계".

**원활한 연속** — 요약 후 히스토리를 클리어하고 현재 기기 컨텍스트가 포함된 새 시스템 프롬프트를 재주입. "중단이 없었던 것처럼 계속"이라는 지시와 함께 요약을 사용자 메시지로 추가.

**별도 압축 모델** — 요약에 저렴/빠른 모델 사용 가능 (예: Haiku로 요약, Opus로 실행).

</details>

<details>
<summary><b>스크린샷 주석 시스템</b> — LLM 공간 추론을 돕는 4계층 시각 레이어</summary>

에이전트가 화면을 읽을 때 스크린샷에 최대 4개 주석 레이어 요청 가능:

1. **좌표 그리드** — 200px 간격 참조선
2. **요소 경계** — 색상 구분 직사각형: 초록=클릭 가능, 파랑=스크롤 가능, 주황=편집 가능, 회색=기타. 각 `[ID]` 표시
3. **텍스트/클래스 라벨** — 요소 클래스명과 텍스트 내용
4. **클릭 마커** — 작업 후 탭 위치를 정확히 표시하여 검증

모든 좌표는 원본 화면 공간 사용 — 변환은 내부에서 처리.

</details>

<details>
<summary><b>다중 프로바이더 LLM 추상화</b> — 하나의 인터페이스, 세 가지 다른 API</summary>

모든 프로바이더가 통합된 `Flow<LLMEvent>`를 방출하지만, 내부에서는 상당한 API 차이를 처리:

- **Claude** — Anthropic 네이티브 SSE 스트리밍. 메시지 역할 정규화 및 연속 동일 역할 메시지 병합. 확장 사고 지원.

- **Gemini** — 도구 사용 시 비스트리밍 강제 (`thoughtSignature`가 스트리밍에서 불안정). 서명은 후속 메시지에서 에코백 필수.

- **OpenAI** — 선택적 속성을 nullable 타입으로 변환하고 모든 속성을 필수화. 모델이 미사용 파라미터에 `null`을 명시적으로 전송하도록 강제.

</details>

<details>
<summary><b>3단계 텍스트 입력</b> — 캐스케이드 폴백으로 범용 텍스트 입력 호환</summary>

1. **내장 입력기** (FoxTouchIME) — 보이지 않는 InputMethodService. WebView, Flutter, 커스텀 컨트롤에 가장 안정적. **Android 10+ 클립보드 읽기 제한 면제**.

2. **ACTION_SET_TEXT** — 포커스 노드에 대한 표준 접근성 액션.

3. **클립보드 붙여넣기** — 최후의 수단: 클립보드에 쓴 후 `ACTION_PASTE` 발송.

IME 자동 전환: 활성화 → 텍스트 커밋 → 이전 입력 방식 복원, 약 1.5초.

</details>

<details>
<summary><b>안전 및 권한 시스템</b> — 단계별 위험 수준과 런타임 모드 전환</summary>

각 도구가 위험 수준(낮음/중간/높음)을 선언. 사용자는 승인 요청에: 허용, 항상 허용, 모두 허용(현재 턴만 YOLO 모드), 거부로 응답 가능. `confirm_completion`은 작업 완료 전 반드시 확인.

</details>

<details>
<summary><b>계획 모드</b> — 실행 전 제한된 관찰 단계</summary>

복잡한 작업에서 에이전트는 읽기 전용 도구와 계획 도구만 사용 가능한 계획 모드에 진입. 구조화된 계획을 작성하여 사용자 검토에 제출, 승인 후 상호작용 도구 잠금 해제. 계획은 재사용 가능한 스킬로 저장 가능.

</details>

<details>
<summary><b>오버레이 아키텍처</b> — 시스템 오버레이 서비스에서 Jetpack Compose 렌더링</summary>

플로팅 패널은 `ComposeView` + `TYPE_APPLICATION_OVERLAY`, 커스텀 `LifecycleOwner`로 렌더링. 에이전트 상태에 따라 포커스 모드를 동적 전환. 스크린샷 전 오버레이를 숨기고 50ms 대기 후 캡처.

</details>

<details>
<summary><b>클립보드 접근</b> — Android 클립보드 제한을 3단계로 우회</summary>

Android 10+ 클립보드 읽기 제한에 대해: IME 경로(제한 면제) → 접근성 서비스 컨텍스트 → 투명 Activity(일시적으로 윈도우 포커스 획득, 2초 내 완료).

</details>

### 다운로드

Release APK는 약 **9 MB**에 불과합니다.

### 요구 사항

- Android 11+ (API 30)
- 접근성 서비스 권한
- 오버레이 (다른 앱 위에 표시) 권한
- 지원되는 LLM 제공자의 API 키 (1개 이상)

### 라이선스

모든 권리 보유. 이 프로젝트는 아직 오픈소스 라이선스로 배포되지 않았습니다.
